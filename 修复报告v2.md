
### 2.3 统一API客户端

创建了 `web/static/js/api-client-v2.js`，功能：

1. **自动适配多种响应格式**:
   - `{success: true, data: {...}}`
   - `{code: 200, data: {...}}`
   - `{code: 0, data: {...}}`
   - 直接返回 `{...}`

2. **统一错误处理**:
   - 自动显示空状态提示
   - 请求超时15秒
   - 控制台错误日志

3. **便捷方法**:
   - `api.get(endpoint)` - GET请求
   - `api.post(endpoint, data)` - POST请求
   - `loadMetrics([...])` - 批量加载指标
   - `showEmptyState()` - 显示空状态
   - `hideEmptyState()` - 隐藏空状态

## 四、使用方法

### 4.1 启动诊断工具

双击运行 `诊断工具v2.bat`，检查：
- 后端服务是否运行
- API端点是否可用
- Mock数据是否已清理
- 空状态提示是否添加

### 4.2 启动后端服务

```bash
cd AGI_FIX
start-backend.bat
# 或
python -m uvicorn api.main:app --reload --port 8000
```

### 4.3 启动前端服务

```bash
start-frontend.bat
# 或
cd web
python -m http.server 3000
```

### 4.4 使用统一API客户端

在HTML页面中引入：

```html
<script src="/static/js/api-client-v2.js"></script>
```

使用示例：

```javascript
// 加载单个数据
await loadData('/system/metrics', 'cpu-value', '--', d => d.cpu);

// 批量加载
await loadMetrics([
    {endpoint: '/system/metrics', elementId: 'cpu-value'},
    {endpoint: '/system/metrics', elementId: 'memory-value'},
]);

// 直接使用
const data = await api.get('/dashboard/stats');
document.getElementById('total').textContent = data.total_agents ?? '--';
```

## 六、后续建议

### 6.1 确保后端API可用

检查 `/api/v1/` 下的路由是否都已注册。如果发现404错误，需要在 `api/main.py` 中添加路由注册。

### 6.2 统一后端响应格式（可选）

如果需要，可以统一后端返回格式为：

```python
{
    "code": 200,
    "message": "success",
    "data": {...}
}
```

### 6.3 添加真实数据源

当后端连接真实数据库后，数据会自动显示，不再显示"暂无数据"。
